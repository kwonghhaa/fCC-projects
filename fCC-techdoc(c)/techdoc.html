Introduction to SQL and Power BI

SQL

Structured Query Language (SQL) is a domain-specific language used to manage data, especially in a relational database management system (RDBMS). It is particularly useful in handling structured data, i.e., data incorporating relations among entities and variables.

Power BI

Microsoft Power BI is an interactive data visualization software product developed by Microsoft with a primary focus on business intelligence. It is part of the Microsoft Power Platform.

SQL and its Commands

SQL is a programming language that you use to retrieve data from the databases. Query - it allows the user to describe desired data, leaving the database management system (DBMS) to carry out planning, optimizing, and performing the physical operations necessary to produce that result as it chooses.

Here are the important commands in SQL

  • SELECT
  • UPDATE
  • DELETE
  • INSERT INTO
  • CREATE DATABASE

MySQL

MySQL Server

MySQL is a relational database organizes data into one or more data tables in which data may be related to each other; these relations help structure the data. SQL is a language that programmers use to create, modify and extract data from the relational database, as well as control user access to the database.

Run SQL in MySQL

  1. Select the database you want to work on.
  2. Type in the SQL commands to retrieve data from the database you selected.

Power BI

Microsoft Power BI

Power BI is a collection of software services, apps, and connectors that work together to turn various sources of data into static and interactive data visualizations.

Data may be input by reading directly from a database, webpage, PDF, or structured files such as spreadsheets, CSV, XML, JSON, XLSX, and SharePoint.

How to connect your MySQL to Power BI?

Power BI desktop has a built-in integration with MySQL, it is not difficult to work on.

  1. Download the Power BI from Microsoft.
  2. Click on "Get Data" and select a "Database" option then choosr MySQL database.
  3. For the new user, create a new user within the database to get the read permissions.
  4. Choose the option for DirectQuery or Import.
  5. Select the tables you want to import into your data model.
